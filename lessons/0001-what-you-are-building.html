What You Are Building

You are building a small Python system that teaches agent thinking from first principles, then uses that same runtime to help people find good open source repositories to contribute to.

The plain-English version

First, we build the smallest possible agent loop: it looks at input, decides what to do, does it, and returns a result. Then we add messages, state, events, and streaming so the agent is easy to inspect in the terminal.

After that, we apply the same runtime to a real problem: ranking GitHub repositories by fit, interest, activity, and contribution potential.

What the lessons are teaching

  1. Chapter 1 explains the agent loop.
  2. Chapter 2 explains messages, state, and results.
  3. Chapter 3 explains events and streaming.

The core idea

observe -> decide -> act -> return result

Later, that becomes:

observe -> decide -> emit events -> act -> final output

Why this matters

If the runtime is clear, the GitHub opportunity agent becomes easier to build, easier to debug, and easier to explain to someone with no background.

That is why we start with the runtime shape first and the GitHub domain second.

Primary source

Read the OpenAI Agents SDK results docs to see the pattern we are borrowing: streamed events during the run, then final output after the stream finishes.

For the data models, read the Pydantic models docs.
