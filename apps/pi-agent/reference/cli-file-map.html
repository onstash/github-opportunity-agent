CLI File Map

A first working CLI should have a small number of files with sharp boundaries. If one file owns too many layers, the app becomes hard to extend.

Suggested Layout

apps/pi-agent/
  src/
    cli.ts
    models.ts
    types.ts
    tools.ts
    sources/
      oss.ts
      jobs.ts
    normalize/
      oss.ts
      jobs.ts
    ranking/
      rank.ts
    render/
      terminal.ts

Ownership

  • cli.ts: orchestration entrypoint
  • models.ts: Pi models collection and selected model
  • types.ts: shared app types like Opportunity
  • tools.ts: Pi tool definitions only
  • sources/*: raw GitHub fetching
  • normalize/*: raw payload to shared type
  • ranking/rank.ts: scoring and sorting
  • render/terminal.ts: stream and final output display

Rule Of Thumb

When a file starts mixing fetch, normalize, rank, and render logic, split it.
