Agent Decision Protocol

A compact reference for the first explicit decision layer in the Pi-inspired agent. This layer sits between loop control and tool execution.

Why It Exists

Without a decision object, the runtime hides its choices inside branching control flow. With a decision object, the runtime can say what it is about to do before it does it.

Minimal First Shape

type AgentDecision =
  | {
      kind: "call_tool";
      toolName: "search_oss" | "search_jobs";
      query: string;
    }
  | {
      kind: "stop";
      reason: StopReason;
    };

Decision Flow

loop iteration
  -> inspect runtime state
  -> decide next action
  -> if call_tool: execute it
  -> if stop: emit completion and exit

State The Decision Reads

  • the latest query
  • candidate tool names
  • already executed tool names
  • iteration count
  • stop rules

Why This Matters

  • makes the runtime inspectable
  • makes testing easier
  • creates a clean place to add eval later
  • creates a clean place to read memory later

What Comes Later

type AgentDecision =
  | { kind: "call_tool"; ... }
  | { kind: "summarize"; ... }
  | { kind: "write_memory"; ... }
  | { kind: "stop"; ... };

The first version should stay small. Start with only call_tool and stop.
