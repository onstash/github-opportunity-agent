Normalization Rules

A normalizer is a translation layer. Its job is not to rank. Its job is to turn source-specific facts into shared comparison signals.

Input And Output

Raw source payload -> normalize -> Opportunity

The input can be messy and source-specific. The output must be stable and predictable.

Normalizer Responsibilities

  • choose a title and summary
  • extract organization and URL
  • merge topics or skills into one list
  • convert raw dates into recency-oriented signals
  • convert source trust hints into confidence-oriented signals

What A Normalizer Should Not Do

  • run the final ranking algorithm
  • make complex user-specific tradeoffs
  • call the model
  • format terminal output

Useful Rule Of Thumb

Normalize facts into signals, then let the ranker decide what those signals mean for this user.

Common Signal Mappings

  • updatedAt -> recencyScore
  • labels like good first issue -> effortScore
  • official org source -> confidenceScore
  • topics or skills -> later relevance checks
