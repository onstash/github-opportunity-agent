Tool Normalization Map

Two tools can return very different raw payloads. The agent becomes manageable only when both are translated into one common opportunity format.

OSS Tool Raw Signals

  • repository name
  • repository URL
  • issue title
  • issue labels
  • stars and recent activity
  • language and topics

Job Tool Raw Signals

  • organization or team name
  • hiring post URL
  • role title
  • role summary
  • required skills
  • recency and credibility hints

Shared Output

type Opportunity = {
  kind: "oss" | "job";
  title: string;
  url: string;
  organization: string;
  summary: string;
  topics: string[];
  relevanceScore: number;
  recencyScore: number;
  effortScore: number;
  confidenceScore: number;
};

Rule Of Thumb

If a raw field cannot help ranking or explanation, do not expose it in the first normalized type.

Pipeline

tool payload -> normalize -> score -> explain -> display
