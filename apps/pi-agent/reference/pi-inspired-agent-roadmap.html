Pi-Inspired Agent Roadmap

A compact map of what the current workspace already has, what is partially built, and what still needs to exist before this becomes a working Pi-inspired agent.

Already Learned

  1. CLI-first agent shape
  2. Pi runtime skeleton
  3. first tool layer
  4. normalization boundary
  5. first explainable ranker
  6. end-to-end file layout

Built In Code Today

  • buildRuntimeInput(...) creates runtime input.
  • toLinearMessages(...) exposes ordered runtime messages.
  • streamRuntime(...) emits structured runtime events.
  • executeRuntime(...) reconstructs a final result from the stream.
  • tool calls are deterministic and inspectable.

Still Missing Before “Working Agent”

  1. Multi-step loop: continue, call tool, summarize, stop.
  2. Decision protocol: a first-class internal assistant decision shape.
  3. Loop controls: maxSteps, repeat detection, stop rules.
  4. Per-step eval: did the last action improve the run?
  5. Working memory: one current-run markdown file.
  6. Episodic memory: one markdown file per run.
  7. Semantic and procedural memory: durable learned facts and heuristics.
  8. Context-doc loading: stable markdown guides loaded into runtime context.

Recommended Build Order

1. multi-step runtime loop
2. internal decision shape
3. stop rules and loop safety
4. step-by-step eval records
5. working-memory.md
6. episodic-memory/*.md
7. semantic-memory/*.md and procedural-memory/*.md
8. context docs loaded into each run

Three States To Remember

current state:
  deterministic streamed tool runner

  bounded multi-step runtime

target state:
  memory-backed Pi-inspired agent with eval
