Opportunity Agent Architecture

A compact blueprint for the first CLI-first Pi AI agent that compares open source and job opportunities in one flow.

Core Loop

user goal -> build context -> choose tools -> fetch signals -> normalize -> rank -> explain -> print result

Pi AI Pieces

  • models: provider and model registry
  • context: system prompt, messages, tools
  • stream(): event stream for visibility
  • Tool: typed actions with validated parameters

App Pieces

  • profile: skills, interests, target roles
  • sources: OSS and jobs adapters
  • normalizer: shared opportunity shape
  • ranker: scoring plus explanation

Shared Opportunity Shape

{
  kind: "oss" | "job",
  title: string,
  url: string,
  organization: string,
  summary: string,
  topics: string[],
  signals: {
    recency: number,
    relevance: number,
    effort: number,
    credibility: number
  }
}

MVP Source Strategy

Use GitHub-native signals first. This is partly an inference from the mission rather than an explicit Pi feature.

  • OSS opportunities: repositories, good-first issues, help-wanted issues, active maintainers
  • Job opportunities: hiring issues, discussions, repository READMEs, organization profiles, and later external adapters

Why CLI First

  • Fastest path to a working feedback loop
  • Easy to inspect stream events and tool calls
  • Easy to snapshot outputs for tests
  • Leaves UI decisions for later
