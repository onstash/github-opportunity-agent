Lesson 2

Build the Pi CLI Skeleton

Now we move from architecture to runtime shape. The goal is not a finished agent yet. The goal is a minimal Pi AI CLI that can run, stream, and stay inspectable.

Quick Correction

Your first answer was directionally right: a shared type works because OSS and jobs mostly differ by kind. The deeper reason is that one shared type lets you reuse the same normalization, ranking, and explanation pipeline.

Your third answer needed one correction: the terminal is only the display surface. The Pi AI feature that makes the runtime visible is the event stream from models.stream(...).

The Five-Part Skeleton

  1. Create a models collection and pick one model.
  2. Define typed tools for OSS and job search.
  3. Build a Context with prompt, messages, and tools.
  4. Run models.stream(model, context) and print events.
  5. Normalize, rank, and print the final opportunities.

Minimal Skeleton

import { Type, type Context, type Tool } from "@earendil-works/pi-ai";
import { builtinModels } from "@earendil-works/pi-ai/providers/all";

const models = builtinModels();
const model = models.getModel("openai", "gpt-4o-mini")!;

const tools: Tool[] = [
  {
    name: "search_oss",
    description: "Find open source opportunities on GitHub",
    parameters: Type.Object({
      query: Type.String()
    })
  },
  {
    name: "search_jobs",
    description: "Find job opportunities related to GitHub orgs or repos",
    parameters: Type.Object({
      query: Type.String()
    })
  }
];

const context: Context = {
  systemPrompt: "You are an opportunity agent.",
  messages: [{ role: "user", content: "Find TypeScript OSS and jobs", timestamp: Date.now() }],
  tools
};

This shape is based on the Pi AI README quick start and tools examples.

The Important Runtime Loop

const stream = models.stream(model, context);

for await (const event of stream) {
  switch (event.type) {
    case "start":
      console.log("run started");
      break;
    case "text_delta":
      process.stdout.write(event.delta);
      break;
    case "tool_call":
      console.log("tool requested", event.toolName);
      break;
  }
}

This loop matters because it separates what the model is doing right now from the final ranked output you print at the end.

What Not To Do Yet

  • Do not add a database yet.
  • Do not add long autonomous loops yet.
  • Do not add many external job boards yet.
  • Do not rank raw OSS and job data before normalization.
